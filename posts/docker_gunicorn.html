[Python] Flask 的 Gunicorn 的 Dcoker 包装模板


将 Flask 包装成 Gunicorn 的 Docker Container

Posted by Yufan Zheng on May 18, 2018

在 Python Web 的开发过程中，经常的我们会因为图方便，使用 flask 这种非常简单并且容易上手的 web server,但 flask 有一个很致命的缺点就是是阻塞的 server，即是说，如果有两个人同时 调用 server 的request 的话，server 就会卡住了，返回不了任何东西，为了防止 server 阻塞，一般的解决办法要么是使用 gevent 在外面包一层，或者是使用 gunicorn。

Gevent 的 server 实际上仍然还是一个单一 server，但是不再是同时处理就会卡掉了，而是有一个队列的处理，所以处理会变慢，但不会卡。Gunicorn 则会将原先的 server 复制几个，做 到真正的并行处理。

创建一个简单的 flask app

在 requirements.txt 中中加入

flask
eventlet
gunicorn

创建一个简单的 app.py 文件，注意，这里的 host 一定要设定成 ‘0.0.0.0‘，否则可能会有问题。

from flask import Flask

app = Flask(__name__)


@app.route('/index')
def index():
    return 'Hello, world'


if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)

文件结构

在正式的写各种脚本之前，我们首先来看一下项目的文件结构

| - bin
  | - create_image.sh
  | - run_container.sh
| - conf
   | - gunicorn.conf
   | - supervisor.conf
| - scripts
   | - flask.sh
   | - gunicorn.sh
   | - kill.sh
   | - restart.sh
| - Dockerfile
| - app.py
| - requirements.txt

启动 flask 的脚本 flask.sh

#!/usr/bin/env bash

cd ../
export FLASK_APP=app.py
# export FLASK_DEBUG=FALSE
flask run --port=5000 --host=0.0.0.0

这个脚本会将 app.py 文件中的 app 到处出来，用 flask command 来启动 server。这可以当做一个测试脚本，当你的server 部署上去的时候， 如果碰到问题，想知道是自己的server 本身有问题，还是gunicorn 的问题，就需要一个单机测试的脚本在这儿，启动它，看是否 OK。

启动 gunicorn 的脚本 gunicorn.sh

#!/usr/bin/env bash

# Kill previous process
bash kill.sh
# Start Process
cd ../
# gunicorn -c conf/gunicorn.conf app:wise_influencer_app
gunicorn -c conf/gunicorn.conf app:app

当 flask 单进程 OK 的时候，就可以考虑 gunicorn 的多进程了，这个里面有提到一个 kill 的 脚本。

#!/usr/bin/env bash

# Process Name
PROCESS=app
# Kill previous process
for pid in $(ps aux | grep $PROCESS | awk '{print $2}')
do
  if [ "$pid" != "" ]
  then
    echo "killing previous process: $pid"
    kill -9 $pid
  fi
done

kill 会杀死之前的进程，然后再开始创建。

重启 gunicorn 的脚本 restart.sh
